## вҡЎ The Problem

рҹ”• Your Copilot Studio bots run on top of large language models. Microsoft updates those models silently. When they do, your bot's behaviour shifts вҖ” subtly or dramatically вҖ” with zero warning. Accuracy drops. Tone changes. Topics misfire. **You find out from a support ticket, not a dashboard.**

## рҹҺҜ The Solution

рҹӣЎпёҸ LLM Drift Tracker watches every bot you care about, around the clock. The moment a model version change is detected in Dataverse, it fires the Copilot Studio Eval API, pulls the results, runs an LLM analysis of the metric delta, and emails you a clean side-by-side report вҖ” all before your users notice anything.

> рҹҡ« No pass/fail verdicts &nbsp;В·&nbsp; рҹҡ« No automated rollbacks &nbsp;В·&nbsp; рҹҡ« No changes to your bots &nbsp;В·&nbsp; рҹ‘ҒпёҸ Pure, unobtrusive observation

### рҹ”‘ Step 2 вҖ” App registration

рҹӘӘ The agent uses **delegated auth** вҖ” it calls the Eval API as you, not as a service. A Microsoft requirement for the Eval API.

```mermaid
sequenceDiagram
    actor Admin
    participant Portal as рҹҸӣпёҸ Azure Portal
    participant PP as вҳҒпёҸ Power Platform API

    Admin->>Portal: рҹ“қ New App Registration
    Portal-->>Admin: рҹ”‘ client_id + tenant_id
    Admin->>Portal: вһ• Add delegated permissions
    Note over Portal,PP: CopilotStudio.MakerOperations.Read + ReadWrite
    Admin->>Portal: вң… Grant admin consent
    Portal-->>PP: рҹҹў Permissions active
```

1. рҹҢҗ [portal.azure.com](https://portal.azure.com) вҶ’ **Azure Active Directory** вҶ’ **App registrations** вҶ’ **New registration**
2. рҹ“қ Name: `copilot-eval-agent` В· Account type: **Single tenant** вҶ’ **Register**
3. рҹ“Ӣ Note the **Application (client) ID** and **Directory (tenant) ID**
4. рҹ”җ **API permissions** вҶ’ **Add a permission** вҶ’ **APIs my organization uses** вҶ’ search `Power Platform API`
5. вң… **Delegated permissions** вҶ’ tick `CopilotStudio.MakerOperations.Read` + `ReadWrite`
6. рҹӣЎпёҸ **Grant admin consent for [tenant]** вҶ’ confirm

## рҹ©ә Troubleshooting

| рҹҡЁ Symptom | рҹ”§ Fix |
|---|---|
| `0 bot(s) tagged #monitor` | рҹҸ·пёҸ Add `#monitor` to bot description вҖ” Copilot Studio вҶ’ Settings вҶ’ Details |
| `no test sets found` | рҹ§Ә Create a test set вҖ” Copilot Studio вҶ’ bot вҶ’ Evaluation tab |
