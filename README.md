## ‚ö° The Problem

Microsoft updates the large language models powering your Copilot Studio bots **silently and without notice**. When a model is swapped, your bot's behaviour changes ‚ÄĒ accuracy drops, tone shifts, topics misfire. You have no visibility. You find out from a support ticket, not a dashboard.

## ūüéĮ The Solution

VARION runs a persistent background agent called **ASHOKA** that watches every bot you care about, around the clock. The moment a model version change is detected in Dataverse, ASHOKA fires the Copilot Studio Eval API, scores every test case, compares the results against the last known-good run, runs an LLM analysis of the delta, and emails you a clean side-by-side report ‚ÄĒ before your users notice anything.

---

## ūüß≠ Philosophy

> **ASHOKA observes. Humans decide.**

ASHOKA is a pure observer. It has no ability to roll back a model, modify a bot, or take any corrective action. Its only job is to surface the truth of what changed and when, with enough data for a human to make an informed decision.

This is deliberate. Automated rollbacks of AI systems carry their own risks. VARION gives your team the signal about what changed and by how much ‚ÄĒ the decision is always yours.

- ūüöę No pass/fail verdicts that auto-trigger action
- ūüöę No automated rollbacks or model changes
- ūüöę No writes to Dataverse or Copilot Studio
- ūüĒ≠ Pure, unobtrusive observation and reporting

### Step 2 ‚ÄĒ App registration

The agent uses **delegated auth** ‚ÄĒ it calls the Eval API as you, not as a service principal. This is a Microsoft requirement for the Eval API.

1. [portal.azure.com](https://portal.azure.com) ‚Üí **Azure Active Directory** ‚Üí **App registrations** ‚Üí **New registration**
2. Name: `copilot-eval-agent` ¬∑ Account type: **Single tenant** ‚Üí **Register**
3. Note the **Application (client) ID** and **Directory (tenant) ID**
4. **API permissions** ‚Üí **Add a permission** ‚Üí **APIs my organization uses** ‚Üí search `Power Platform API`
5. **Delegated permissions** ‚Üí tick:

| Permission | Purpose |
|---|---|
| `CopilotStudio.MakerOperations.Read` | Trigger and retrieve evaluation results |
| `EnvironmentManagement.Environments.Read` | Auto-discover Power Platform environments |

6. **Grant admin consent for [tenant]** ‚Üí confirm

> Both permissions are on the same `Power Platform API` resource ‚ÄĒ a single sign-in covers everything.
> Without `EnvironmentManagement.Environments.Read`, Load Environments will fail ‚ÄĒ manual environment entry still works.
> Environment IDs can also be found manually in make.powerapps.com ‚Üí Settings ‚Üí Session details.

### Step 3 ‚ÄĒ Create test sets

> Without test sets, ASHOKA has nothing to evaluate and will skip the bot.

```
Copilot Studio ‚Üí your bot ‚Üí Evaluation tab ‚Üí New test set
```

Add 10‚Äď20 utterances covering your bot's main topics. ASHOKA discovers and runs all test sets automatically.

| `force_eval` | Triggered manually from dashboard or CLI |

## ūüĒź Auth

All API calls (Eval API, Power Platform environmentmanagement, Power Platform Inventory) share one MSAL `PublicClientApplication` and one `SerializableTokenCache` file. Device flow runs once; all subsequent calls acquire silently via the cached refresh token.

| Scenario | Behaviour |
|---|---|
| First run | MSAL device code flow ‚ÄĒ browser sign-in, token cached |
| Subsequent runs | Silent refresh ‚ÄĒ no user interaction |
| Token expired | Agent emails admin a new device code, polls for 15 min |
| Docker / Azure | Mount `msal_token_cache.json` on the shared volume |

## ūü©ļ Troubleshooting

**Tuning via `config.json`:**

| Key | Default | What it controls |
|---|---|---|
| `watch_interval_seconds` | `120` | How often the watcher polls Dataverse for model changes |
| `eval_poll_timeout_seconds` | `1200` | Max time to wait for Power Platform to finish running evals |
| `eval_poll_interval_seconds` | `20` | How often to ping the Eval API during polling |
| `log_level` | `"INFO"` | Log verbosity ‚ÄĒ `"DEBUG"` for full detail, `"ERROR"` for quiet |

| Symptom | Fix |
|---|---|
| `0 bots found` | Check `monitoredBots` in `config.json` ‚ÄĒ or rerun Setup to re-pick |
| `no test sets found` | Create a test set in Copilot Studio ‚Üí bot ‚Üí Evaluation tab |
| Nothing in dashboard | Use **‚Ė∂ Force Eval** on the bot detail page, or run `.\drift.bat eval` |
| LLM 401 error | Check `LLM_API_KEY` in `.env` ‚ÄĒ key must match the endpoint |
| `MSAL auth failed` | Re-authenticate via Setup ‚Üí Authentication ‚Üí Sign In |
| 403 on Load Environments | App registration needs `EnvironmentManagement.Environments.Read` with admin consent ‚ÄĒ sign out and back in after granting |
| SMTP failed | Office 365: `smtp.office365.com:587` ‚ÄĒ password in `.env` as `SMTP_PASSWORD` |
| Container exits immediately | `docker compose logs varion-agent` ‚ÄĒ likely missing volume or env var |
| Timeline empty | Run a force eval ‚ÄĒ it will write the first events to `data/events.jsonl` |
| Logs tab empty | Start the agent ‚ÄĒ `data/agent.log` is created on first run |
| Memory warning in log | Agent RSS grew >50% from baseline ‚ÄĒ check for large Dataverse or eval API responses being held in memory across cycles |
| Log file missing after restart | File rotates at 5 MB; check `agent.log.1` for previous session |
