## вҡЎ The Problem

рҹ”• Your Copilot Studio bots run on top of large language models. Microsoft updates those models silently. When they do, your bot's behaviour shifts вҖ” subtly or dramatically вҖ” with zero warning. Accuracy drops. Tone changes. Topics misfire. **You find out from a support ticket, not a dashboard.**

## рҹҺҜ The Solution

рҹӣЎпёҸ LLM Drift Tracker watches every bot you care about, around the clock. The moment a model version change is detected in Dataverse, it fires the Copilot Studio Eval API, pulls the results, runs an LLM analysis of the metric delta, and emails you a clean side-by-side report вҖ” all before your users notice anything.

> рҹҡ« No pass/fail verdicts &nbsp;В·&nbsp; рҹҡ« No automated rollbacks &nbsp;В·&nbsp; рҹҡ« No changes to your bots &nbsp;В·&nbsp; рҹ‘ҒпёҸ Pure, unobtrusive observation

2. рҹ“қ Name: `copilot-eval-agent` В· Account type: **Single tenant** вҶ’ **Register**
3. рҹ“Ӣ Note the **Application (client) ID** and **Directory (tenant) ID**
4. рҹ”җ **API permissions** вҶ’ **Add a permission** вҶ’ **APIs my organization uses** вҶ’ search `Power Platform API`
5. вң… **Delegated permissions** вҶ’ tick `CopilotStudio.MakerOperations.Read` + `ReadWrite`
6. рҹӣЎпёҸ **Grant admin consent for [tenant]** вҶ’ confirm

## рҹ“Ӣ Event log вҖ” `data/events.jsonl`

Every agent action is recorded as a JSON line:

```jsonl
{"ts":"2026-04-18T14:14:32+00:00","event":"force_eval","detail":"force_eval.trigger detected"}
{"ts":"2026-04-18T14:14:35+00:00","event":"eval_start","botName":"Safe Travels","detail":"Eval triggered"}
{"ts":"2026-04-18T14:17:17+00:00","event":"eval_complete","botName":"Safe Travels","detail":"pass 80%  В·  avg 52.5  В·  REGRESSED","passRate":0.8,"avgScore":52.5}
{"ts":"2026-04-18T14:17:18+00:00","event":"regression","botName":"Safe Travels","detail":"Regression in: CompareMeaning.passRate"}
```

| Event | Logged when |
|---|---|
| `cycle_start` | Poll cycle begins (scheduled or forced) |
| `model_change` | Model version shift detected |
| `eval_start` | Eval API call initiated |
| `eval_complete` | Eval finished вҖ” includes pass rate, avg score, verdict |
| `eval_timeout` | Eval polling timed out |
| `eval_no_sets` | No active test sets found вҖ” bot skipped |
| `regression` | One or more metrics regressed vs previous run |
| `improvement` | One or more metrics improved |
| `stable` | No drift вҖ” model unchanged |
| `force_eval` | force_eval.trigger file detected |
| `error` | Unhandled exception during cycle |

## рҹ”җ Auth reference

All three token types (Eval API, BAPI, Dataverse) share one `PublicClientApplication` and one `SerializableTokenCache` file. Device flow is initiated once; subsequent calls acquire silently via the cached refresh token.

| Context | Token acquisition |
|---|---|
| First run | MSAL device code flow вҖ” browser sign-in, token cached to file |
| Subsequent runs | Silent refresh from cache вҖ” no user interaction |
| Token expired | Agent emails admin a new device code, polls for 15 minutes |
| Docker / Azure | Mount `msal_token_cache.json` on shared volume вҖ” both containers share the session |

## рҹ©ә Troubleshooting

| Symptom | Fix |
|---|---|
| `0 agent(s) found` | Check `monitoredBots` in `config.json` вҖ” or run setup to re-pick bots |
| `no test sets found` | Create a test set in Copilot Studio вҶ’ bot вҶ’ Evaluation tab |
| `no drift detected` | Run `.\drift.bat eval` (Win) / `./drift eval` (bash) to force evals regardless of model change |
| LLM analysis shows 401 | Check `LLM_API_KEY` and `LLM_BASE_URL` in `.env` вҖ” the key must match the endpoint region |
| `drift` not found in PowerShell | Use `.\drift.bat dashboard` or add project dir to `$env:PATH` |
| `MSAL auth failed` | Re-authenticate via the Setup page вҖ” click **Sign In** to trigger device-code flow |
| BAPI 401 on Load Environments | Ensure the app registration has admin consent for `service.powerapps.com` вҖ” add delegated permission scope `https://service.powerapps.com/.default` and grant admin consent |
| `SMTP test failed` | Office 365 uses `smtp.office365.com:587` вҖ” password goes in `.env` as `SMTP_PASSWORD` |
| Container exits immediately | Run `docker compose logs drift-agent` вҖ” likely a missing volume or env var |
| Timeline shows no events | Run `.\drift.bat eval` вҖ” it will append the first real events to `data/events.jsonl` |
